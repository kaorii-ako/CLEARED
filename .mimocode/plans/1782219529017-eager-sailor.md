## FocusFlight Design Reference
- **Globe**: Dark globe with animated airport tags — CLEARED uses SVG equirectangular instead of 3D globe (simpler, lighter)
- **Boarding pass**: Physical ticket look with tear-off stub — same pattern, cream paper, perforated line, barcode
- **Takeoff**: Perspective runway animation — same concept
- **Cruise map**: 3D map with flight path — CLEARED uses MapLibre 2D (lighter, cross-platform)
- **Window view**: Airplane window porthole — same concept, CLEARED uses real satellite tiles
- **Landing**: LANDED stamp — same rubber stamp overlay
- **History**: Flight log cards — same mini boarding pass pattern
- **Key difference**: CLEARED uses real OpenSky data (real callsigns, real routes, real time) vs FocusFlight's fictional routes

## Key Implementation Details

### Data Pipeline
1. App opens → `fetchCruisingFlights()` → cache 60s → display on globe
2. User selects flight → `fetchFlightRoute(icao24)` → resolve with `resolveFlightDuration()`
3. Boarding pass shows real data → "Cleared for departure" starts session
4. `useFlight.ts` polls every 30s → updates position, altitude, speed
5. Landing: countdown=0 OR on_ground:true → save to localStorage → show LandedPass

### OpenSky Rate Limiting
- `fetchCruisingFlights`: called once, cached 60s
- `fetchFlightRoute`: throttled 1 req/1.2s (sequential through 12 flights)
- `pollAircraft`: every 30s during session
- Null return: keep last known state, SIGNAL LOST badge, 3 nulls → landing detection

### localStorage
- `cleared:sessions[]` — SessionData[]
- `cleared:totalKm` — cumulative km
- `cleared:mapStyle` — 'dark' | 'satellite' | 'classic'
