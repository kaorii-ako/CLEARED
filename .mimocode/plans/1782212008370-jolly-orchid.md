**Goal**: Central state management with reducer pattern.

| File | Content |
|------|---------|

## Data Preparation

### airports.json
Download `https://ourairports.com/data/airports.csv`, filter:
- `type` in ['large_airport', 'medium_airport']
- Has `iata_code` and `ident` (ICAO code)
- Keep: icao, iata, name, city, country, latitude_deg, longitude_deg, elevation_ft
- Output as JSON array, ~2000 entries

### aircraft-types.json
- Source: https://github.com/Jonty/icao-aircraft (public domain)
- Map: `{[icao24_hex]: {manufacturer: string, type: string}}`
- This is ~200KB trimmed — too large to create manually, use the source data

### world.svg
- Natural Earth 110m simplified land polygons
- Equirectangular projection (lon -180→180, lat -90→90)
- Dark navy fill (#1a2035) on near-black bg (#0d1220)
- ~60KB, continents only

## Key Risks & Mitigations

| Risk | Mitigation |
|------|-----------|
| OpenSky API slow/unavailable on first load | Graceful loading state with retry; mock data fallback for development |
| MapLibre WebGL issues in Electron | Set `maplibre-gl/dist/maplibre-gl.css` import; enable `forceWebGL1` if needed; add `--enable-gpu` to Electron flags |
| Linux compositor transparency issues | Detect with `win.isTransparent()`, fall back to opaque #08090d background |
| aircraft-types.json too large for manual creation | Download from Jonty/icao-aircraft repo or generate with a script |
| OpenSky rate limiting | Strict 1 req/1.2s for routes, 60s cache on bulk, backoff on 429 |
